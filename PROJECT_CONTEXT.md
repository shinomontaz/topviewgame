## 🏗️ Architecture & Technical Decisions

### **Core Architecture: ECS (Entity Component System)**
- **Движок:** Ebiten v2.8.8 (Go game engine)
- **ECS библиотека:** bytearena/ecs v1.0.0
- **Компоненты:** Position, Renderable, Health, MeleeWeapon, Armor, Name, UserMessage
- **Системы:** Player, monsters, Render, Attack, HUD, UserLog, Animation, Cursor

### **Ключевые модули:**
1. **Game Loop** (`main.go`) - основной игровой цикл
2. **Level Generation** (`level.go`) - процедурная генерация подземелий
3. **Components** (`components.go`, `component_*.go`) - игровые компоненты
4. **Systems** (`system_*.go`) - игровые системы
5. **State Management** (`state/`) - система состояний для анимаций
6. **Controllers** (`controller/`) - обработка ввода пользователя
7. **Pathfinding** (`astar.go`) - алгоритм поиска пути

### **Визуальная система:**
- **Tile-based рендеринг** (32x32 пикселя)
- **FOV система** с затемнением невидимых областей
- **Шейдеры** для визуальных эффектов (grayscale для неисследованных областей)
- **Sprite animations** с системой состояний

## 🔧 Implementation Patterns & Standards

### **ECS Patterns:**
- Компоненты как структуры данных без логики
- Системы обрабатывают сущности с определенными компонентами
- Теги для группировки сущностей ("players", "monsters", "renderables")

### **Game State Management:**
- Turn-based система с состояниями PlayerTurn/EnemyTurn
- State machine для анимаций персонажей (Stand, Idle, Attack, Death, Walk)

### **Code Organization:**
- Один файл на систему/компонент
- Глобальные переменные для ECS компонентов
- Интерфейсы для абстракции (Renderable, controllable, stater)

## 🐛 Known Issues & Technical Debt

### **Архитектурные проблемы:**
1. **Глобальные переменные** для ECS компонентов (positionC, renderableC, etc.)
2. **Тесная связанность** между системами
3. **Отсутствие конфигурации** - хардкод значений в коде
4. **Нет системы сохранения/загрузки**
5. **Ограниченная система событий**

### **Геймплейные ограничения:**
1. **Простая боевая система** - только ближний бой
2. **Нет системы предметов/инвентаря**
3. **Ограниченные типы монстров** (только Skeleton и Zombie)
4. **Один уровень** - нет перехода между этажами
5. **Нет системы прогрессии персонажа**
